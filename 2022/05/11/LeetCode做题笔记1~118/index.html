1.1 两数之和[简单][1]

题目要求：给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target  的那 两个 整数，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
你可以按任意顺序返回答案。

（暴力解法）解法一：逐个对比每两个元素之和是否等于目标元素的值。

（哈希表）解法二：现给出更好解法，利用unordered_map容器生成哈希表mp，最坏情况下，也仅需一次遍历就可以找到答案，具体思想是，哈希表,记录访问过的元素信息，key保存记录的元素的值，value保存下标。遍历元素nums[i]时，检查哈希表中是否存在key为target - nums[i]的元素，如果有，这个元素和nums[i]之和便是target。该元素下标就是mp[target-nums[i]],直接返回即可。如果没找到，将nums[i]的信息保存至哈希表，继续遍历。

1.2 承最多水的容器[中等][11]

题目说明：给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。
找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。
返回容器可以储存的最大水量。
说明：你不能倾斜容器。

（暴力解法）解法一：设左侧指针从0逐渐指向n-2，每次左侧指针增加1对应一次循环。每次循环中固定左侧指针，右侧指针从n-1逐渐指向左侧指针的后一位。 循环初始维护右侧指针指向的最大值temp_high，若右侧指针指向的值小于temp_high，则不计算本次盛水量，直至出现比temp_high大才计算。若右侧指针指向的值大于本次循环左侧指针指向的值，计算承水量并退出本次循环。max维护计算的最大成水量。左侧指针同理，只有当左侧指针指向的值大于temp_Lhigh才开始这轮循环，否则略过。

上述方法时间效率差，下面引入更好的方法。

（双指针法）解法二：分别定义左右指针left与right，它们分别指向数组两端。由于盛水面积由短边决定，所以若左/右侧指针指向的值较小，移动左/右指针向中间靠拢，计算盛水面积，max维护最大盛水面积。直至left与right指针重叠，停止移动。返回max。

1.3 三数之和[中等][15]

题目说明：给你一个包含n个整数的数组nums，判断nums中是否存在三个元素a，b，c，使得a + b + c = 0?
请你找出所有和为0且不重复的三元组。
*注意：答案中不可以包含重复的三元组。

解法一：为了方便找到答案，首先对容器元素进行排序。这样便于找到不重复的答案。从左至右遍历容器中的元素，将其设为target，若找到另外两个元素满足三个元素之和为0，则找到该target下的解。这样每个target下的三数之和，就转变为给定目标值的两数之和。进行两数之和查找时，左侧指针初始位置从target元素后一个位置，不断递增至容器末尾。右侧指针初始位置是容器末端。将侧指针与右侧指针指向元素的和与-target比较，如果大于则右侧指针减小直至与左侧指针重叠或两数之和等-target，这是将元素推入目标容器中，为了让答案不会出现重复。对于两层循环，如果枚举的数据与上一个数据相同，跳过该次循环。

1.4 最接近的三数之和[中等][16]

题目说明：给你一个长度为n个整数的数组nums，和一个目标值target。请你从nums中选出三个整数，使它们的和与target最接近。
返回这三个数的和。
假定每组输入只存在恰好一个解决。

（双指针法）解法一：该题类似15题三数之和，同样可以使用双指针的方法。首先对容器进行排序，第一个指针从左至倒数第三个位置遍历整个数组。第一个指针每移动一次，开始一次循环。该循环中，第二个指针指向第一个指针后一位置，第三个指针指向数组最后一个位置，然后比较这三个指针指向元素之和与目标元素的距离，如果比历史最小距离小，更新最终返回值。如果三数之和大于目标值，右侧指针减小，如果三数之和小于目标值，左侧指针增大。

1.5 删除有序数组中的重复项[简单][26]

题目说明：给你一个 升序排列 的数组 nums ，请你 原地 删除重复出现的元素，使每个元素 只出现一次 ，返回删除后数组的新长度。元素的 相对顺序 应该保持 一致 。由于在某些语言中不能改变数组的长度，所以必须将结果放在数组nums的第一部分。更规范地说，如果在删除重复项之后有 k 个元素，那么 nums 的前k 个元素应该保存最终结果。将最终结果插入 nums 的前 k 个位置后返回 k 。
不要使用额外的空间，你必须在原地修改输入数组 并在使用 O(1) 额外空间的条件下完成。

解法：由于数组为升序，所以只要相邻的两个元素不相同，就说明出现了新的不重复的值。定义j记录不重复的值的个数，当出现不重复的值的时候，令nums[j]等于这个不重复的值，同时j的值增加一，这样就能保证数组的前j个元素的值都是不重复的。最终返回j。

1.6 移除元素[简单][27]

题目说明：给你一个数组nums和一个值val，你需要原地移除所有数值等于val的元素，并返回移除后数组的新长度。不需要使用额外的数组空间，你必须仅使用O（1）额外空间并原地 修改输入数据
元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

解法：与[26]题思想类似。从头至尾遍历整个数组，定义j初始为0。当遍历的值不是val时，先令nums[j]等于当前遍历值，然后有j++。这样最终j记录的就是除去val外其余值的个数。并且数组的前j个元素都是不为val的元素。

1.7 下一个排序[中等][31]

题目说明：整数数组的一个排列就是将其所有成员以序列或线性顺序排列。

  • 例如，arr = [1,2,3] ，以下这些都可以视作 arr 的排列：[1,2,3]、[1,3,2]、[3,1,2]、[2,3,1]。

整数数组的 下一个排列 是指其整数的下一个字典序更大的排列。更正式地，如果数组的所有排列根据其字典顺序从小到大排列在一个容器中，那么数组的 下一个排列 就是在这个有序容器中排在它后面的那个排列。如果不存在下一个更大的排列，那么这个数组必须重排为字典序最小的排列（即，其元素按升序排列）。

  • 例如，arr = [1,2,3] 的下一个排列是 [1,3,2] 。
  • 类似地，arr = [2,3,1] 的下一个排列是 [3,1,2] 。
  • 而 arr = [3,2,1] 的下一个排列是 [1,2,3] ，因为 [3,2,1] 不存在一个字典序更大的排列。
    给你一个整数数组 nums ，找出 nums 的下一个排列。
    必须 原地 修改，只允许使用额外常数空间。

解法一：将nums中的数字组合看作一个整数，本题的目标简而言之，就是寻找这些数字可以组合出的、仅比当前nums大的一个数字组合。如果没有比当前nums更大的数字组合，说明当前nums中的数字组合出了可能的最大值，按照题意，要将nums的数字组合变为最小的数字组合。明白了题意后，这道题就清晰很多了。我们从nums的最后一个元素为基准元素，然后从右至左遍历nums,寻找比基准元素更小的元素。找到目标元素后，记录目标元素下标位置为sub_a。记录基准元素的元素下标为sub_b。紧接着开始下一次遍历，每次遍历基准元素左移一位，直至移动到sub_a+1。每次遍历时，如果新发现的目标值的下标比sub_a大，更新sub_a为该目标元素下标，同时更新基准元素下标sub_b。这样，经过遍历之后，我们得到两个下标sub_a和sub_b。比较容易能够理解，仅比当前nums数字组合更大的数字组合，一定有nums[sub_a] == nums[sub_b]。而sub_a前的元素排列与当前nums保持一致。于是，为了保证得到仅比当前nums更大的数字组合，我们只需交换nums[sub_a]与nums[sub_b]，然后将nums[sub_a]的后续元素按照升序排列即可。

1.8 搜索旋转排序数组[中等][33]

题目说明：整数数组 nums 按升序排列，数组中的值 互不相同 。 在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 在下标 3 处经旋转后可能变为 [4,5,6,7,0,1,2] 。 给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。
你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

（二分搜索）解法一：因为旋转后的数组分为两个有序的部分，直观上感觉使用二分搜索比较困难。所以我们的基本思路就是，首先找到旋转点的位置，这样的话，旋转点前以及旋转后的两部分都是有序的，然后确定搜索元素可能在哪一个部分。再对该部分进行二分搜索即可。寻找旋转点的过程也是使用二分搜索的思想：定义left和right初始为数组两端。mid = (left+right)/2。比较中点的值与两端的值，此时有两种情况。1.中点值大于左端点值。则说明旋转点在中点与右端点的区间。并且更新左右端点与中点。2.中点值小于左端点值，说明旋转点在左端点与中点的区间。更新左右端点与中点。重复上述过程，直到中点等于左端点，此时右端点就是旋转点。找到旋转点之后，比较target与数组两端的值，如果target >nums[0]。则target有可能在旋转点左侧有序数组中。如果
target<nums[n-1]。target有可能在旋转点右侧的有序数组中。其余情况nums中不存在target。找到可能的区间后，只需在可能的区间范围进行二分搜索即可。此处不在赘述。

1.9 在排序数组中查找元素的第一个和最后一个位置[中等][34]

题目说明：给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。
如果数组中不存在目标值 target，返回 [-1, -1]。
你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

（Carl的解法）解法一：根据题目O(log n)的要求，显然要用二分查找。我们先进行特殊情况的处理，当数组第一个元素比目标元素大，或者最后一个元素比目标元素小，又或者数组为空时。数组中没有目标元素。此时返回[-1,-1]即可。接下来我们使用二分查找的思想来分别寻找左右边界。这里我们先假设数组中有目标元素，首先查找右边界，分别定义left和right为数组左右两端。mid为left到right区间的中心点。搜索条件是left <= right，搜索右边界的逻辑是：

  1. 如果nums[mid]比target大，说明右边界肯定在mid的左侧。更新right = mid - 1。
  2. 如果nums[mid]不大于target。说明右边界肯定在mid的右侧。更新left = mid + 1。更新右边界rightborder = left。 当left>right时，停止搜索，此时rightborder - 1就是右边界值。之所以右边界时rightborder - 1。是因为最后一次搜索时，有left == right，满足搜索的条件，并且此时满足上述条件2，即nums[mid] == target。此时右边界值应该是right。执行了left = mid + 1，rightborder = left。由于执行语句前right ==mid == left。故最终得到的rightborder减去1才是真正的右边界的下标值。搜索左边界的思想与之类似，不在赘述。需要特别说明的是数组中没有目标元素的情况。当数组中没有target时，左右边界会指向数组中同一个元素。所以仅当rightborder - leftborder > 1时，即数组中至少有一个target时，返回{leftborder + 1,rightborder - 1}。否则返回{-1,-1}。

1.10 搜索插入位置[简单][35]

题目说明：给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。
请必须使用时间复杂度为 O(log n) 的算法。

解法： 本题使用二分从查找即可，首先处理特殊情况，当数组中首元素大于target时，插入首元素即可。当数组中最后一个元素小于target时，在最后位置插入即可。排除特殊情况后，target可能在数组中，使用二分查找，找到target时返回下标即可。
当元素中没有target时，二分查找left的最后停留的位置，是数组中第一个target大的元素，即要插入位置，此时返回left即可。

1.11 有效的数独[中等][36]

题目说明：请你判断一个 9 x 9 的数独是否有效。只需要 根据以下规则 ，验证已经填入的数字是否有效即可。

  1. 数字 1-9 在每一行只能出现一次。
  2. 数字 1-9 在每一列只能出现一次。
  3. 数字 1-9 在每一个以粗实线分隔的 3x3 宫内只能出现一次。

注意：

  • 一个有效的数独（部分已被填充）不一定是可解的。
  • 只需要根据以上规则，验证已经填入的数字是否有效即可。
  • 空白格用 '.' 表示。

（暴力解法）解法一：我们使用board[i][j]来定位到坐标为（j，i）的元素。为了让描述更为符合直觉，以下用（x，y）代替（j，i）。判断是否为数独的前两条很简单，以检查元素在某一行是否重复为例，我们只需要检查（x+1，y）到（8，y）
这些元素是否与该元素相等，若相等，返回false即可。之所以不用检查（0，y）到（x-1，y）这些元素。是因为这些元素在（x，y）元素之前已经检验过了，函数能够进行到这里，必然没有返回false。即（x，y）与这些元素不等。没有必要检查。检查列元素是否重复与之类似。现在检验第3个性质。我们令x_num = (x / 3) * 3。y_num = (y / 3) * 3。不难理解横坐标为（x_num，x_num + 2）。纵坐标为（y_num，y_num + 2）的区间就是元素所在3x3区间。检验区间内元素是否与（x，y）元素相等即可。由于我们已经进行了横纵方向的判别。故只有区间内横纵坐标不等于x与y的元素有必要与（x，y）元素对比。

（哈希表）解法二：利用哈希表来求解这道题更为简单，能够将时间复杂度减低到O(1)。具体思想如下：分别初始化三个
哈希表，其中第一个哈希表用于存储每种元素在该行出现的次数，一个哈希表用于存储每种元素在该列存储的次数，最后一个
哈希表用于存储每种元素在该3X3区域出现的次数。设置完哈希表后，开始遍历。每当检测到一个不是'.'的元素后，更新哈希表。
因为元素都是char类型的。所以令board[i][j] - '0' - 1得到应该在哈希表存储的下标位置。更新哈希表，然后检查更新完之后
，哈希表该位置元素是否大于1。若大于1，说明该元素在某行/某列/某3X3区域不止一个，返回false。否则继续遍历。直到遍历
81个元素。

1.12 组合总和[中等][39]

题目说明：给你一个 无重复元素 的整数数组 candidates 和一个目标整数 target ，找出 candidates 中可以使数字和为目标数 target 的 所有 不同组合 ，并以列表形式返回。你可以按 任意顺序 返回这些组合。 candidates 中的 同一个 数字可以 无限制重复被选取 。如果至少一个数字的被选数量不同，则两种组合是不同的。
对于给定的输入，保证和为 target 的不同组合数少于 150 个。

（回溯算法）解法一：本题可以归类为分类问题，由于目标数字的个数不是确定的，单纯使用循环语句很难解决此问题，故考虑使用回溯算法。回溯算法有三个组成部分。

  1. 终止条件
  2. 函数参数
  3. 单层搜索逻辑

对于终止条件，明显当搜索到的数字之和大于等于target，应该终止。对于函数参数，题目给出的数组candidates及目标值显然target是必要的，另外递归搜索的起始位置也是必要的，我们定义为start_index。对于本题，还要定义sum用于存储当前已经搜索到的元素之和，以方便检验他们是否满足要求。每次递归时，我们将元素存储到数组之中，同时sum更新。然后进入下一层递归。紧接着递归过程之后，是一个回溯的过程，这也是回溯算法的核心思想。
为了保证答案不出现重复，我们将下一次递归的起始位置，设置为当前搜索的位置。

1.13 组合总和Ⅱ[中等][40]

题目说明：给定一个候选人编号的集合 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。
candidates 中的每个数字在每个组合中只能使用 一次 。
注意：解集不能包含重复的组合。 

（回溯法）解法一：本题与[39]组合总和不同之处在于，数组中的每个数字只能使用一次。其余要求一样。故解题思路就是在在[39]题的代码上进行修改，由于每个数字只能使用一次，故下一次递归时的起始位置不能与当前位置一致，需要改为当前位置的下一个位置。但即便如此，最后的答案可能会出现重复的情况，例如输入为[1,1,2],3这种情况。第一个1和2,第二个1和2组成的数组，结果都为3。这显然是不符合题目给出的不包含重复组合的要求。所以要进行去重的操作，这也是本题的难点及核心。回想以前做过的去重的操作逻辑，一般是满足if(i > 0 && nums[i] == nums[i - 1])条件时，结束本次循环。对于本题目而言，如果把递归组合的过程看作一个树的形式，要想不能包含重复的组合，必须满足如下条件：

  • 同一树层的数字不能重复。

但如果我们在单层搜索时，令if(i > 0 && nums[i] == nums[i - 1]) continue;。会造成组合中不能选取重复的数字，这是不符合题意的。例如输入为[1,1,1,1,1,6],4。由此我们提炼出本题目的第二点要求：

  • 同一树枝的数字可以重复。

由此我们更改去重的条件为if(i > start_index && nums[i] == nums[i - 1]) continue;。这样处理的话。单层搜索时出现重复的元素就会跳过循环，避免了同一数层出现重复的数字。而由于单层搜索时递归调用自身，下一层递归时的起始位置为下一个位置。此时即使该位置数字与上一个位置的数字重复，但由于本次递归从下一个位置开始，是可以选用该元素的。由此我们使得同一树枝可以有重复的数字。

（Leetcode官解二）解法二：解法一分为两部完成题目，分别为：1.线序遍历二叉树；2.按照线序遍历序列展开二叉树。那么能否遍历二叉树的同时完成链表的展开？要想搞清楚这个问题，首先要明白为什么在一般的先序遍历中不能直接展开二叉树。如果在一般的线序遍历中，直接展开二叉树，按照题目要求展开二叉树，所以会丢失右子树的信息，具体来说。我们一直遍历左子树，按照线序遍历不断将左子树的元素作为根节点的后继，根节点右指针域指向它，这将会使得原来右子树的信息丢失。故无法做到。如果我们对线序遍历做一些修改，就可以完成做到线序遍历的同时展开二叉树，我们都知道，线序遍历的顺序为{根节点，{左子树},{右子树} }。要想展开的时候，不丢失右子树的信息，需要在展开前保存右孩子信息。使用栈可以解决该问题。具体为，我们定义pre、cur为线序遍历中前后两相邻结点。其中cur为当前遍历结点。pre初始为空，定义栈sta保存结点，初始把root压栈，之后开始循环，循环终止条件为sta为空。进入循环，首先把栈顶元素赋予cur，然后将pre的左孩子置空，右孩子指针域指向cur,对pre的操作只有pre不为空才执行。之后依次把cur的右、左孩子结点压栈。最后pre = cur。先后把cur的右、左孩子结点压栈保证了保留右孩子结点信息的同时，能按照先序遍历的顺序遍历二叉树。

（Leetcode官解三）解法三：通过总结，我们可以得到，对于二叉树的某一结点node，如果其左子树为空，则该节点不需要做处理。当node的左子树不为空时，寻找接下来先序遍历的下一个结点next,由于node左子树不为空，故next即为node的左孩子结点，我们寻找node左子树中最右侧的结点，设为parents，让其作为node右子树的前驱结点。之后让parents右指针域指向node的右孩子结点，之后node左孩子结点置空，右孩子指针域指向next，如此，对于node结点的处理完毕了，我们继续处理next，令node = next。重复上述循环，可以展开二叉树，上述过程文字理解起来比较难，建议自己画画图，就很容易理解了。

（Leetcode官解二）解法二：解法一空间复杂度为O(N)。现在给出空间复杂度为O(1)的更优解。与解法一不同的是，解法一是遍历第N层的同时，连接第N层，本解法遍历第N层的同时，连接第N + 1层，具体为。对于第N + 1层的节点，其需要连接的相邻两个节点，为如下两种情况之一：

  1. 连接1：两节点有相同的双亲结点。
  2. 连接2：两节点的双亲结点不同，且前一个结点为其双亲结点的右节点，后一个结点为其双亲的左节点。

于是我们遍历第N层的元素node，如果node有子节点。首先其子节点需要进行上述连接1,如果node不为该层最后一个元素，即node->next不为空，其子节点要进行连接2。具体来说，连接1代码为node->left->next = node->right。连接2为node->right->next = node->next->left。当node判断并执行完这两种连接后，有node = node->next。当遍历完改成元素，即node == NULL时。node要变为下一层第一个元素，具体实行可以每层开始遍历时，预先保存下一层的首元素。当node没有孩子时，根据完美二叉树的定义，此时所有结点已经连接完毕。

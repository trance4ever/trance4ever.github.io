java学习笔记

2024-11-02

记录java学习的过程，由于我之前学过几年C++，这个笔记更多侧重于java与C++的不同点。

java语言与C++的不同点

java的文件名称

java的文件名称一定要与类名保持一致，而C++则对文件的名称没有特殊要求。

linux系统的编译与运行命令

对于C++代码来说，编译器是g++，对应的编译命令是g++，linux中可以直接运行编译完成的文件
但是对于java代码，需要使用javac命令就行编译，生成的.class文件需要使用java命令运行，例如：
javac helloworld.java
java helloworld

java的项目结构

java的项目，分为项目、模块、包、类四个层级， 需要依次创建这些，其中包的名称应当用com.name.appname的形式命名。
可以看到，java项目与C++项目的结构是不同的，C++项目没有这么明确的层级，在vs studio开发，项目仅有头文件和源文件的区别。

char数据类型

与C++不同，java中char数据类型占用两个字节，这是由于C++中的char类型存储的是ascii码，而java存储的是unicode码，这大概也是java的char数据类型能够表示中文汉字的原因。另外在java中布尔类型用boolean表示，不同与C++。

long数据类型

在java中，整数默认为int类型，所以当右值为数字且范围超出int类型时，IDE会报错，需要在数字的末尾加上L或者l，这样表示这个数字是long数据类型的。

float数据类型

与C++不同，在书写float类型的数据时，java需要在数据后面加上一个f或F。如float f = 1.1f。这一点跟C++是不同的。这是因为java默认小数为double类型。

String数据类型

java中的字符串类型为String。需要注意的是首字母大写。String对象是不可变字符串对象，每次以双引号定义的字符串对象。都会放到堆内存的字符串常量池中，并且相同
字符串只会存放一份。通过new方式创建的字符串对象，每new一次都会产生一个新的对象存放在堆内存。由于String类型保存的是字符串的引用，所以当字符串对象==运算时比较的是地址，而非字符串的内容。

表达式的自动类型转换

在java中，char、byte和short数据类型自动转换为int类型进行运算。

类型检查

在java中，如果左值的数据类型占用字节数小于右值的数据类型占用的字节数，编译会报错。而在C++中这一操作是被允许的。

+运算符

在java中，+根据实际情况，为加法运算符或链接符号，当+两端连接的是字符串和其他类型时，输出的结果是字符串的拼接。

java的逻辑运算符

java的逻辑运算符!仅可用于boolean类型对象，并且不同于C++非0即为真，java语言的判断语句严格要求boolean类型的数据。

switch

java的switch支持字符串类型。

方法

java的方法这一概念，基本等同于C++的函数，定义形式也是C++基本相同。下方给出了一个实例。

类

java中，一个类文件只有一个类可以用public修饰，且该类的名称要与文件名称相同，此外，尽管认为java没有指针的概念。java定义一个对象实际保存的是该对象的引用，当一个堆中的对象没有任何
引用时，会认为这块内存时垃圾内存，在java的垃圾回收机制下会回收，C++的智能指针与之类似。
除此之外，java中的类都是开辟在堆区，而C++中使用new关键字开辟的对象才一定在堆区。直接定义的对象保存区域视情况而定。
类的static修饰符与C++中的功能类似。

this

java的this，等同于C++中的*this,通过这个关键字可以访问类中的成员。

static

使用static修饰的成员是属于类的成员。java通过类名.对象名可以访问到类的对象，C++则是类名::对象名。

java类的访问权限

java类的成员有四种访问权限，分别为public、protected、缺省和private。他们的访问权限依次递减。除了private修饰的成员，其他成员都可以被同一软件包里的其他类访问，java中缺省修饰的成员不能被继承。
除此之外，父类中的同名对象只能在子类中通过super.方法进行访问。

继承

java的类只能单继承，但是可以多层继承，即一个类只能继承一个对象，但是可以有B类继承A类，C类再继承B类。java的类都会继承object类，可以使用object类提供的一些方法。

方法重写

java子类可以重写父类的方法，在方法定义上方用@Override修饰，可以检查子类的方法参数是否和父类完全一致。
子类重写的方法的权限必须不小于父类的方法。
子类重写的方法的返回值类型一致，范围不大于父类的方法。
C++中的重写，指的是派生类重写父类继承的虚函数。

构造方法

java构造方法调用顺序与C++构造函数调用顺序相同，先调用父类，在调用子类。并且在子类中显式的调用父类的构造器，形如super(...)来调用父类的有参构造器，来应对父类未提供无参构造器的场景。这类似于C++的成员初始化列表构造父类的匿名对象。
java中，通过this可以调用兄弟构造器。形如this(...)。

多态

java的多态与C++基本相同：

  • 都是父类指针指向子类对象（前面提到过java对象保存的是对象在内存中的地址，也即指针）
  • 父类定义一个方法
  • 子类重写父类提供的方法
  • 指针访问父类提供的方法，根据实际指向的对象的数据类型，调用子类重写的方法

类型校验

java中，同样可以通过(转换数据类型)来进行强制类型转换，可以通过instanceof检查数据类型，形如x instanceof 数据类型。返回一个boolean类型数据，与C++的dynamic_cast<指针类型>用法类似。

final关键字

final有如下用法：

  1. final修饰类，则该类不能被继承
  2. 使用final修饰方法，则方法不能被重写
  3. 使用final修饰变量，则变量只能被赋值一次，当final修饰引用类型时，类似于C++的指针常量，指向的内存地址不能变，引用的数据可以变。
  4. 使用static final修饰的成员变量为常量，等同于C++中的宏定义。

抽象修饰符abstract

用abstract修饰一个类，则该类为抽象类，无法实例化，用abstract修饰一个方法，则该类为一个抽象方法，拥有抽象方法的类为抽象类，换句话说，如果子类继承了父类的抽象类，并且没有重写，遵循上述原则，子类也为抽象类。
与C++关联在于

  1. java的抽象类对应C++的纯虚类，C++的纯虚类是指具有纯虚函数的类，对应java的抽象方法。
  2. C++不需要显式的用一个修饰符定义纯虚函数，只需要让虚函数= 0即可，形式上更加灵活
  3. 抽象类（纯虚类）都不可以实例化

内部类

java和C++都可以定义内部类，但是这两种语言实例化内部类的方法不同，C++通过作用域，可以控制内部类的实例化，内部类可以独立于外部类存在。而java的内部类的实例化必须依托于外部类。对于在堆区开辟内部类的语法，java为：外部类.内部类 对象名称 = new 外部类构造方法.new 内部类构造方法。而对于C++，为：外部类::内部类 对象名称 = new 外部类::内部类构造函数。不难看出，C++给出了更为灵活的对象的访问机制，而java则更为严谨。
除此之外，java在内部类中可以通过外部类.this.成员名称来访问外部类的成员，而C++需要手动计算偏移量来得到外部类的指针，再访问外部类，形如outer* po = (outer*)((char*)this - offsetof(outer, in))。
其次，java可以定义静态成员类。实例化形式为外部类.静态内部类 对象名称 = new 外部类.静态内部类构造方法。静态内部类不能访问外部类的成员（这里指对象特有的，非类特有的，即静态成员）。

泛型编程

java支持泛型编程，形如修饰符 class 类名<A,B,C...>。尖括号内表示模板类型的个数。
对于泛型接口，每个实现类需要确定一种数据类型。如public class A implements XX<类名>。
对泛型方法，定义形式为：修饰符 <A,B...> 返回值类型 方法名(参数列表) 方法体。
通配符?代表所有类型，这个有点类似于C/C++的万能指针类型void*。
除此之外，还可以对类型进行一定的限制，形式为<A extends 类B>。此时要求类型A为类B或其子类。
结合上述两点，? extends A要求一个A或其子类的类型，? super A要求一个A或其父类的类型。
java的泛型编程与C++关联：

  1. C++的模板编程支持基本类型，java只支持对象类型。
  2. C++的模板函数或类只在实例化的时候编译，而java的泛型编程在编译过后被擦除，从这个角度讲，他们都是作用在编译阶段。

java小记

JVM、JRE与JDK

JVM全称为java virtual machine，即java虚拟机，是java程序运行的地方。
JRE全称为java runtime environment，即java运行环境，包含JVM和核心类库，核心类库为java为程序员提供的程序。
JDK全称为java development kit，即为java开发工具包，包含上述所有内容和java开发工具。

java跨平台的原理

java程序一次编译、处处运行。这是因为java程序是在JVM上运行的，而不同平台的JDK各自拥有适配该平台的JVM，所以可以实现这个特性。这也是java深受市场欢迎的一个原因。

接口基础

java提供接口，接口只能有成员变量和成员方法，并且成员变量是常量，成员方法是抽象方法。
接口的优势之一就是一个实现类可以继承多个接口。
当接口继承的多个接口存在签名冲突，或继承多个接口的类中存在签名冲突时，则该方法无法实现。
一个类继承了父类以及实例化了接口，当它们当中存在同名方法时，子类优先使用父类的方法。
一个类实现了多个接口，当多个接口中有同名默认方法是，实现类可以通过重写该方法避免冲突。

接口新特性

jdk8中为接口提供了三种新特性，接口中可以定义默认方法、私有方法和静态方法。其中默认方法只能由接口的实现类对象访问，私有方法只能由默认方法调用，静态方法可以通过接口名称访问。

clone方法

java的Object类提供了clone方法，想要使用clone方法，需要如下两个步骤：

  1. 类实现Cloneable接口
  2. 类重写clone方法

clone方法是值传递的，也即：

  1. 对于基本数据类型，拷贝值
  2. 对于字符串常量类型，拷贝字符串地址
  3. 对于引用类型，拷贝引用地址

所以，默认clone方法是浅拷贝，要想实现深拷贝，需要在clone方法完成引用类型的拷贝过程。

jdk8新增-本地时间类

此前jdk提供的时间类API存在线程不安全等问题，jdk8新增了LocalDate、LocalTime和LocalDateTime类，下面介绍这些类的常用API。
now()：返回系统当前时间类的对象。
LocalDate常用API：

  • getYear()、getMonthValue、getDayofMonth()：取得年月日信息
  • getDayOfYear()：获取当前为一年中的第几天
  • getDayOfWeek()：获取现在为星期几
  • withYear()、withMonth()、withDayOfMonth()、withDayOfYear()：直接修改某个信息，并返回一个新的修改时间后的对象。愿对象信息不变
  • plusYears()、plusMonths()、plusWeeks()：把某个信息加多少，返回一个新的对象，原对象不变
  • minusYears()、minusMonths()、minusWeeks()：把某个信息减多少，返回一个新的对象，原对象不变
  • of(int year, int month, int day)：获取指定日期的LocalDate对象
  • equals()、isBefore()、isAfter()：比较日期是否相等，是否在某日期之前、是否在某日期之后

LocalTime常用API：

  • getHour()、getMinute()、getSecond()、getNano()：获取时分秒纳秒
  • withHour()、withMinute()、withSecond()、withNano()：修改时分秒纳秒
  • plusHours()、plusMinutes()、plusSeconds()、plusNanos()：时分秒纳秒加上多少
  • minusHours()、minusMinutes()、minusSeconds()、minusNanos()：时分秒纳秒减去多少
  • of(int hour, int minute, int second)：获取指定日期的LocalTime对象
  • equals()、isBefore()、isAfter()：比较时间是否相等，是否在某时间之前、是否在某时间之后

对于LocalDateTime，这个类包含上述两个类的所有内容，包含上述所有同名接口。功能也一样。LocalDateTime可以相互转化为LocalDate和LocalTime对象。
转换方法：LocalDateTime.toLocalDate()、LocalDateTime.toLocalTime()以及LocalDateTime.of(LocalDate date, LocalTime time)

Period可以计算两个LocalDate对象相差的年数、月数、天数。常用方法如下：

  • getYears()：计算间隔年数

  • Duration between()：依据两个时间得到Duration对象

  • toMinutes()：计算隔多少分钟，并返回
  • toSeconds()：计算隔多少秒，并返回
  • toMillis()：计算隔多少毫秒，并返回
  • toNanos()：计算隔多少纳秒，并返回

  2. 自定义仿函数，作为排序函数中添加的谓词
  3. 在需要排序的对象中，重载比较运算符，此方法类似于java的方法一
  4. 排序函数中，定义一个lambda表达式作为排序规则，这个方法与java的方法二类似

lambda表达式

java提供lambda表达式，可以且只能简化函数式接口，即只有一个方法的接口。在C++中，lambda表达式是匿名函数，这两种语言的lambda表达式，在功能上都是快速定义一个方法/函数，可能是java万物皆对象，所以使用lambda表达式仍然需要一个函数式接口。

利用方法引用简化lambda表达式

  • 静态方法引用可以进一步简化lambda表达式，使用规则为：如果lambda表达式只是调用一个静态方法，并且前后参数形式一致，就可以用类名::静态方法名称来使用lambda表达式
  • 实例方法引用与静态方法引用用法相同。使用方法为：实例::实例方法名
  • 特定类型的方法引用适用于lambad表达式只是调用一个实例方法的情况，并且参数列表的第一个参数是方法的主调，而后一个参数作为方法的入参。使用方法为：类型::实例方法名
  • 构造器引用适用于lambda表达式只是创建对象，并且前后参数一致的情况，使用场景存疑，使用方法：类名::new

正则表达式

String的matches方法将字符串与正则表达式进行匹配，正则表达式常用书写规则如下：

  • []中，-表示范围，^表示不匹配项，&&表示或匹配
  • \后面：d表示数字，D表示非数字，s表示空格，S表示非空格，w表示字母、下划线或数字，W表示^\w
  • 数量匹配：X?匹配不超过一次，X*匹配零次或多次，X+匹配一次或多次，X{n}表示正好出现n次，X{n,}表示至少出现n次，X{n,m}表示出现n至m次
  • 其他：.匹配任意单个字符，(?i)忽略大小写，|表示或，()表示分组

正则表达式相关的API：

  • String.replaceAll(String regex, String newStr)：按照正则表达式替换字符串
  • String.split(String regex)：按照正则表达式分割字符串，返回一个字符串数组

异常处理

java中可以自定义异常类，根据异常阶段的不同，可以定义运行时异常类或编译时异常类，这两种使用方法为：

  • 对于运行时异常类，需要继承RuntimeException，之后重写构造方法，之后可以通过throw抛出异常
  • 编译时异常使用方法同上，区别在于运行前提示错误

除此之外，方法后用throws 自定义异常类修饰，可以把异常抛给上层处理。

集合的并发修改异常

当遍历集合的同时，又在删除集合中的数据，程序就会出现并发修改异常。
出现这种原因，是由于删除元素后，可能会导致迭代器失效，即迭代器没有指向正确的元素。

Stream

Stream是JDK8添加的一套API，用于操作数组与集合中的数据。
Stream流的使用步骤：

  • 首先有一个数据源，它可以是数组或集合
  • 从数据源获取Stream流
  • 调用Stream中间方法
  • 调用Stream终结方法

Stream流常用方法

获得Stream方法有：

  • stream()：获得collection的Stream流
  • Arrays.stream(T[] arr)orstream.of(T ... value)：获得数组的Stream流

Stream流的中间方法支持链式编程，即处理完后返回一个数据流，常用中间方法有：

  • filter(predicate<? super T> predicate)：按照自定义规则过滤数据
  • sorted()：按照默认规则排序数据
  • sorted(comparator<? super T> comparator)：按照自定义规则排序数据
  • limit(long maxSize)：限制返回maxSize个数据
  • skip(long n)：跳过n条数据
  • map(Function<? super T, ? extends R> mapper)：获得映射的数据，类似SQL的select
  • distinct()：去除重复元素，类似SQL的distinct，默认比较地址，如果需要去除内容相同的对象，需要重写hashCode()和equals()方法
  • concat(Stream a, Stream b)：把两个流合并为一个

Stream使用终结方法后，不会再返回Stream流对象，常用的终结方法有：

  • forEach(consumer action)：按照自定义规则遍历数据
  • count()：计算流中数据个数，类似SQL中的count
  • max(Comparator<? super T> comparator)：得到特定规则下的最大的对象
  • min(Comparator<? super T> comparator)：得到特定规则下的最小的对象
  • collect(Collector collector)：把流处理后的结果收集到一个指定的集合中去
  • toArray()：把流处理后的结果收集到一个数组中去

Logback

Logback日志框架基于SLF4j接口，包括三个模块：支撑其他两个模块的logback-core模块，完整实现slf4j的logback-classic模块以及提供HTTP访问
的logback-access模块。
想要使用Logback框架，需要在项目中导入三个包：slf4j-api、logback-core和logback-classic。
并且还需要在模块src目录下添加logback.xml来格式化日志输出。
仅需在需要使用日志的类添加日志器：public static final Logger LOGGER = LoggerFactory.getLogger("LogBackTest");，之后调用日志器info()等方法输出日志。

logback.xml

配置输出到控制台：<appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
配置输出到文件：<appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
控制输出格式：<pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
控制日志拆分：<rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
日志级别控制：<root level="INFO">，日志器只会输出日志级别大于level的日志

Thread常用方法

线程安全

多个线程访问共享资源时，可能会出现问题，为此需要使用锁等手段进行同步。
java提供了如下方法来保证线程安全：

  1. 同步代码块：synchronized(锁) {代码块}：建议用共享资源作为锁，如果是实例方法，建议用this作为锁，如果是静态方法，建议使用类名.class作为锁
  2. 同步方法：把访问共享资源的核心方法给上锁，即使用synchronized修饰方法
  3. 利用Lock锁对象手动加锁和解锁

线程通信

Object类提供了等待和唤醒方法，可用与线程通信，这些方法分别为：

  • wait()：线程阻塞，等待唤醒
  • notify()：唤醒被阻塞的单个线程
  • noifyAll()：唤醒被阻塞的所有线程

线程池使用方式一

创建一个线程池，可以通过线程池的实现类ThreadPoolExecutor的构造函数来实现，为：ThreadPoolExecutor(int corePoolSize, int maximumPoolSize, long keepAliveTime, TimeUnit unit, Blockingqueue<Runnable> workQueue, ThreadFactory threadFactory, RejectedExecutionHandler handler)。
上述参数分别为：

  • corePoolSize：核心线程数量
  • maximumPoolSize：最大线程数量
  • keepAliveTime：指定临时线程存活时间
  • unit：临时线程存活的时间单位
  • workQueue：线程池的任务队列
  • threadFactory：线程池的线程工厂
  • handler：指定线程池的任务拒绝策略

线程池常用方法：

  • execute(Runnable task)：处理task
  • Future<T> submit(Callable<T> task)：运行Callable类型任务，并返回Futrue<T>结果
  • shutdown()：任务执行完毕后，关闭线程池
  • List<Runnable> shutdownNow()：立即关闭线程池，并返回未执行的任务

对于计算密集型任务，核心线程数应该为线程数 + 1
而对于IO密集型任务，核心线程数应该为线程数 * 2

线程池使用方式二

Executors是一个线程池的工具类，提供如下方法返回线程池对象：

  • ExecutorService newFixedThreadPool(int nThreads)：创建固定线程数量的线程池
  • ExecutorService newSingleThreadExecutor()：创建只有一个线程的线程池
  • ExecutorService newCachedThreadPool()：新建一个线程数量随任务变化的线程池
  • ScheduleExecutorService newScheduledThreadPool(int corePoolsize)：创建一个线程池，可以实现定时任务

这些方法，实际上还是通过ThreadPoolExecutor创建线程池。

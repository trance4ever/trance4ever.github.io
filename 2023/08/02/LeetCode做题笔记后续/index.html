1.1 杨辉三角Ⅱ[简单][119]

题目说明：给定一个非负索引 rowIndex，返回「杨辉三角」的第 rowIndex 行。
在「杨辉三角」中，每个数是它左上方和右上方的数的和。

（暴力解法）解法一：使用两个数组完成本题，pre表示上一个数组，cur表示当前生成数组，pre初始为{1}，由此我们从杨辉三角第二行还是生成。每一行除了第一个和最后一个元素，都能找到其左上角和右上角的元素，具体为cur[i]的左上角元素为pre[i - 1]，右上角元素为pre[i]。之后两数相加得到数值，由此生成cur。生成完后更新pre = cur，重复上述过程直到生成了rowindex行为止。

1.2 三角形最小路径和[中等][120]

题目说明：给定一个三角形 triangle ，找出自顶向下的最小路径和。
每一步只能移动到下一行中相邻的结点上。相邻的结点 在这里指的是 下标 与 上一层结点下标 相同或者等于 上一层结点下标 + 1 的两个结点。也就是说，如果正位于当前行的下标 i ，那么下一步可以移动到下一行的下标 i 或 i + 1 。

（动态规划）解法一：本题用动态规划比较容易解决，我们直接修改数组triangle，定义triangle[i][j]表示从三角形顶端到下标(i,j)位置的最小路径和。易知三角形第一个元素不用修改，考虑triangle[i][j],到达(i,j)位置的可能是其左上方，也有可能是其右上方，故定义cur为其右上方最小路径和为triangle[i - 1][j]，定义其左上方元素为pre为triangle[i - 1][j - 1],triangle与pre与cur的更小者相加，最终得到到达(i,j)的最小路径和。当左/右上角元素不存在时，定义它为INT_MAX。更新triangle数组后，遍历最后一行元素并返回最小值即可。

1.3 买卖股票最佳时机[简单][121]

题目说明：给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。
你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。
返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

解法一：一次遍历可以解决该问题，定义MAX维护当前遍历天数可获得最大利润。定义temp_min维护当前遍历天数以前的最低股票价格，遍历prices[i]时，MAX取MAX与prices[i] - temp_min的最大值。这样MAX的值即为遍历到prices[i]为止能够获得的最大利润。遍历完数组之后返回MAX即可。

1.4 买卖股票的最佳时机Ⅱ[中等][122]

题目说明：给你一个整数数组 prices ，其中 prices[i] 表示某支股票第 i 天的价格。
在每一天，你可以决定是否购买和/或出售股票。你在任何时候 最多 只能持有 一股 股票。你也可以先购买，然后在 同一天 出售。
返回 你能获得的 最大 利润 。

（LeetCode官方解法一）（动态规划）解法一：定义dp[i][0]表示第i天不持有股票时的最大利润，dp[i][1]表示第i天持有股票时的最大利润。dp[i][1]可第i - 1天不持有股票，第i天买股票，或者第i - 1已持有股票的状态到达，同理，dp[i][0]可由第i - 1天购买股票，第i天卖出股票，或者第i天不持有股票到达。具体要分别取他们两种状态利润的最大值。故有：

  1. dp[i][0] = max(dp[i - 1][1] + prices[i],dp[i - 1][0]);
  2. dp[i][1] = max(dp[i - 1][0] - prices[i],dp[i - 1][1]);

对于初始化，有dp[0][0] = 0,dp[0][1] = -prices[0]。根据上述递推公式，求得的最后一天不持有股票的利润，即为目标答案。

（LeetCode解法二）（贪心算法）解法二：定义(i,j)为第i天买入，第j天卖出的区间，则本题即求若干(i,j)区间产生最大利润之和，定义max(i,j)为该区间最大利润。可以证明max(i,j)等同于把(i,j)分为j - i个长度为1的区间，每个区间求最大利润。故本题可以分割为求n - 1个区间，并求其最大利润。

1.5 最长连续序列[中等][128]

题目说明：给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

（哈希表）解法一：题目明确提出O(n)级时间复杂度，所以暴力解法就不适用了，考虑使用哈希表。定义哈希表mp,键值对为<int,int[2]>，其中键表示元素值，值为数组，其中数组第一个元素表示当前元素所在区间的左边界，数组第二个元素表示当前元素所在区间的右边界。本解法一次遍历可以完成解题，思路如下，遍历数组nums，遍历到nums[i]时，定义nums[i] = cur,哈希表中没有这个元素，就在哈希表中添加这个元素，如果哈希表中已经存在这个元素，跳过避免重复。之后我们寻找当前元素的前/后两个元素，定义为pre/next在哈希表中是否存在，如果存在，更新哈希表中当前元素所在区间的左/右边界值为mp[pre][0]/mp[next][1]。一个新的元素到达，可能有如下三种可能：

  1. 该元素连接哈希表中已知区间的右边界。
  2. 该元素连接哈希表中已知区间的左边界。
  3. 该元素不与哈希表中任意元素相连。
  4. 该元素连接哈希表中两个区间。

对于情况1，对应的是mp[pre]存在，而mp[next]不存在，此时要更新哈希表区间左边界元素的右边界值，有mp[mp[cur][0]][1] = mp[cur][1]。
对于情况2，对应的是mp[next]存在，而mp[pre]不存在，此时要更新哈希表区间右边界元素的左边界值，有mp[mp[cur][1]][0] = mp[cur][0]。
对于情况3，对应的是mp[next]不存在，mp[pre]不存在，此时不必特殊处理。定义mp[cur]的第一二个元素都为cur。
对于情况4，对应的是mp[next]存在，而mp[pre]也存在，此时即要更新哈希表区间右边界元素的右边界值，又要更新哈希表区间右边界元素的左边界值，有mp[mp[cur][0]][1] = mp[cur][1],mp[mp[cur][1]][0] = mp[cur][0]。最终返回哈希表中左右边界差值最大的值 + 1 即为正确答案。
之所以上述过程可以解题，是因为一个新到达元素，只可能是以上四种情况之一，并且因为一个非重复元素如果与区间相连，坑定连接一个区间的边界。于是我们只要每次更新这个元素（区间新的边界），以及该元素不相邻的另一个边界，就可以保证区间的边界元素信息正确。由此可以保证最后结果正确。

（LeetCode官方解法）（哈希表）解法二同样是使用哈希表，现在有一种更为简单的方法，首先用unordered_set容器st，保存nums中元素，这样做可以去除nums中重复元素。之后遍历st容器，定义遍历到元素为num，如果num - 1存在，跳过本次遍历，这样做，最终num为某个区间的左边界。之后不断在容器中寻找num + 1,num + 2,....，同时记录长度。找到的最大区间长度即为目标。本方法每个元素最多只需要遍历两次即可。故为0(n)时间复杂度。

1.6 加油站[中等][134]

题目说明：在一条环路上有 n 个加油站，其中第 i 个加油站有汽油 gas[i] 升。
你有一辆油箱容量无限的的汽车，从第 i 个加油站开往第 i+1 个加油站需要消耗汽油 cost[i] 升。你从其中的一个加油站出发，开始时油箱为空。
给定两个整数数组 gas 和 cost ，如果你可以绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1 。如果存在解，则 保证 它是 唯一 的。

（Leetcode官方解法）解法一：假设从x出发，每一站都加完油，能够到的最远的站是y站，现在说明从[x,y)中的任何一站z出发，都到不了y的下一站。因为从x出发到z站时，汽车油箱中的油量一定大于等于0。如此情形下都尚且到不了y的下一站，而从z出发时油箱中的油为空的情形下就更到不了y的下一站了。所以推论成立。
于是我们从下标0开始遍历加油站，模拟从此地开始环形。如果碰到第一个到不了的加油站y + 1。就从y + 1开始出发，重复上述过程，直至n个加油站都遍历过。如果找到行驶了n个结点的出发点index就返回它，如果遍历后都没找到能行驶n个站点的出发点，就返回-1。

1.7 乘积最大子数组[中等][152]

题目说明：给你一个整数数组 nums ，请你找出数组中乘积最大的非空连续子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。
测试用例的答案是一个 32-位 整数。
子数组 是数组的连续子序列。

解法一：我们注意到，nums存储的是整数，于是，不难得出，连续区间数值的乘积的绝对值是不递减的（此处先不考虑区间内包含零），于是，我们把区间以数值0分割为若干个连续子区间，对于某以子区间，如果区间内包含偶数个负数，该区间最大值即为整个区间的乘积，若该区间包含奇数个负数，则该区间最大值为该区间左端到倒数第一个负数前一个元素之间的乘积，或是该区间末端至该区间第一个负数之后元素区间的乘积。各个区间中的最大值即为最大值。

（Leetcode官方解法）解法二：定义MAX(i)维护已当前遍历下标位置i的连续区间的最大乘积，定义MIN(i)维护最小乘积，定义nums[i]为已遍历元素。则MAX(i)为MAX(i - 1),nums[i],MIN(i - 1) * nums[i]中的最大值，MIN(i)则为上述元素中最小值。遍历当中MAX(i)的最大值即为最大值。

1.8 寻找旋转排序数组中的最小值[中等][153]

题目说明：已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：

  • 若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
  • 若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]

注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。
给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。
你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

解法一：看到时间复杂度要求，不难得出要用二分查找。定义left、right初始分别指向数组两端，之后进行二分查找,比较区间中点mid = (left + right) / 2与区间左边界left,若nums[mid] > nums[left]，说明left至mid这段区间有序，所以最小值位于mid + 1与right区间内，如果nums[mid] < nums[left]，说明mid + 1置right这段区间有序，最小值存在于left至mid - 1这段区间内。若nums[left] == nums[mid]，说明此时区间只有两个元素，并且此时区间右边界就是最小值。每次比较后更新区间边界。直到找到最小值，然后返回。
本题需要特殊情况的处理：数组旋转0次，且数组元素个数大于等于2。此时对应条件nums[left] < nums[right] && nums[left] <= nums[(left + right) / 2]。此时返回nums[left]。

1.9 寻找峰值[中等][162]

题目说明：峰值元素是指其值严格大于左右相邻值的元素。

给你一个整数数组 nums，找到峰值元素并返回其索引。数组可能包含多个峰值，在这种情况下，返回 任何一个峰值 所在位置即可。
你可以假设 nums[-1] = nums[n] = -∞ 。
你必须实现时间复杂度为 O(log n) 的算法来解决此问题。

（Leetcode官方解法）解法一：看到O(log n),想到用二分法，题目相邻元素不相等。定义l和r分别指向区间两端，之后开始二分查找，定义mid为查找区间中点。定义mid_right，如果mid不为区间右边界，则mid_right = nums[mid + 1]，如果为区间右边界，则为INT_MIN。区间取舍为：

  1. 如果nums[mid] < mid_right,说明峰值在mid的右侧，更新l = mid + 1。
  2. 其他情况,峰值在mid以及其左侧，更新r = mid。

最后一定有l指向峰值，故二分搜索完后返回l。

1.10 两数之和Ⅱ-输入有序数组[中等][167]

题目说明：给你一个下标从 1 开始的整数数组 numbers ，该数组已按 非递减顺序排列 ，请你从数组中找出满足相加之和等于目标数 target 的两个数。如果设这两个数分别是 numbers[index1] 和 numbers[index2] ，则 1 <= index1 < index2 <= numbers.length 。
以长度为 2 的整数数组 [index1, index2] 的形式返回这两个整数的下标 index1 和 index2。
你可以假设每个输入 只对应唯一的答案 ，而且你 不可以 重复使用相同的元素。
你所设计的解决方案必须只使用常量级的额外空间。

（双指针法）解法一：定义指针l、r初始分别指向数组左右两端。然后判断：

  • 如果nums[l] + nums[r] == target,返回{l + 1,r + 1}
  • 如果nums[l] + nums[r] > target,此时应有r--
  • 如果nums[l] + nums[r] < target,此时应有l++

根据上述过程，如果答案下标为{i + 1,j + 1}。0 <= i < j <= n - 1。则一定时左指针或右指针先到达i/j，然后另一个指针再到达j/i。

（Leetcode官方解法二）（二分查找）解法二：每次固定一个元素nums[i]，然后再剩余区间[i + 1,n - 1]进行二分查找寻找target - nums[i]。
**tips:**二分查找循环条件l <= r可以解决最后区间为两个元素，而目标元素在右边界的情况。

1.11 多数元素[简单][169]

题目说明：给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。
你可以假设数组是非空的，并且给定的数组总是存在多数元素。

（哈希表）解法一：定义哈希表记录遍历的数字以及其出现的次数，如果该元素出现次数大于n / 2，返回它。

解法二：将所有元素的32位的值累加，如果第i位1的个数大于n / 2，则说明多数元素第i位为1。

（Leetcode官方解法二）解法三：将数组排序，之后数组的中间元素一定时众数元素。

（LeetCode官方解法四）（分治）解法四：将区间分为两段，分别找出每段中的众数，在确定这两个众数哪一个是整个区间的众数。

1.12 轮转数组[中等][189]

题目说明：给你一个数组，将数组中的元素向右轮转 k 个位置，其中 k 是非负数。

解法一：定义n位数组长度，经过三个步骤可以完成操作，具体如下：

  • 按照顺序预先记录数组下标n - k到n - 1的这部分元素到vec。
  • 依次将数组下标n - 1到k的元素用其前k个位置的元素代替。
  • 将数组下标0至k - 1的元素用vec中的元素还原。

（Leetcode官方解法）解法二：不难发现，轮换后，数组最后k % n个元素移至数组前k % n个位置，而数组前n - k % n个元素移至数组末尾，于是首先我们令整个数组反转，使得以上两个区域元素待在正确的位置，然后依次反转数组(0,k - 1)与(k,n - 1)区域的元素，可以得到最终答案。

1.13 打家劫舍[中等][198]

题目说明：你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。
给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

（动态规划）解法一：维护两个变量steal和not_steal，其中steal表示当前房屋偷窃的情况下能够得到的最多金额，not_steal表示当前天不偷窃的情况下能够得到的最多金额，之后遍历数组，遍历第i个房屋时，由于steal一定要在这件房屋偷窃，于是steal等于前i - 1间房屋维护的not_steal加上第i间房屋金额，而not_steal取前i间房屋产生的steal和not_steal中的最大值即可。

1.14 长度最小的子数组[中等][209]

题目说明：给定一个含有 n 个正整数的数组和一个正整数 target 。
找出该数组中满足其和 ≥ target 的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。
题目连接

（滑动窗口）解法一：定义ans维护满足条件的最小区间长度，初始为INT_MAX，定义left维护滑动窗口的最边界，定义right维护滑动窗口的右边界，初始left = right = 0，定义cur维护滑动窗口内元素之和。遍历数组，根据滑动窗口元素之和cur值不同，有如下情况：

  1. cur < target,则滑动窗口右边界右移。
  2. cur >= target,此时先更新ans，之后滑动窗口左边界右移。

直到right到达数组右端点，并且cur < target时，停止遍历。最终返回ans。

（快速排序）解法一：本题考察排序算法，这里首先介绍快速排序，快速排序对于数组nums的区间[left,right]进行排序，基本思想如下：

  • 任取数组中一个数作为pivot，为了方便代码编写可以每次都取待排序区间第一个
  • 定义l和r初始分别指向区间左右两端，之后轮换移动两指针，将所有大于pivot的元素移动到pivot的前面，所有不大于pivot的元素移动到pivot前面
  • 当l和r重合时，该位置即为pivot的位置
  • 排序完成后，对pivot前后两个子区间同样执行快速排序

由于本题只是找出第k大元素，所以快速排序完成后，不必对前后两个区间都进行排序，定义数组一共n个元素，保证排序包含下标n - k的区间即可，如果pivot的下标恰好为n - k，pivot即为第k元素。

（堆排序）解法二：另一种快速排序的算法为堆排序，堆排序以数组形式维护一个堆，本题维护一个大根堆，大根堆可以用完全二叉树表示，并且有如下性质：

  • 左右子树的所有结点小于双亲节点
  • 下标为i的元素双亲下标为(i - 1) / 2
  • 下标为i的元素左孩子下标为i * 2 + 1
  • 下标为i的元素右孩子下标为i * 2 + 2

这里先说明维护堆的过程，大根堆的结点的左右孩子都小于双亲结点，对于一个结点，我们检查它是否大于其孩子结点，如果小于某个孩子结点，交换这两个结点的值，并且递归的维护该孩子结点。
堆排序首先要建堆，即从第一个有孩子结点的结点开始，从后往前的维护堆，这样一个大根堆就形成了，此时堆的根节点为最大元素，于是我们得到最大元素，我们将其放到数组最后，然后对该堆的根节点就行维护，就可以得到第二大元素，将其放到数组倒数第二个位置,…直到重复n - 1此，此时完成排序。
本题是找到第k大元素，所以维护k - 1次大根堆即可。

  • root本身为q或这p节点，其左子树或右子树包含另一节点。

所以我们深度优先遍历二叉树，定义lson表示节点左子树是否包含目标节点，同理rson表示右子树是否包含目标节点。则当上述条件满足其一，即(rson && lson) || ((root == p || root == q) && (lson || rson)))时，此时节点root即为目标节点，本解法的精妙之处在于，除了两节点的最近公共祖先，其余两节点的所有祖先都不满足上述两条件其一。

（Leetcode官方解法二）（哈希表）解法三：此方法最为直观，遍历整个二叉树，利用哈希表记录每个节点的双亲节点。之后从q开始不断访问其祖先节点，并标记以访问节点，然后同样不断访问p的祖先节点，直至跳转到被访问过的节点，该节点就是两节点最近祖先节点。

接下来说明深度优先搜索的解法，我们从结点的出度出发，定义edges记录图中所有的边，定义visited记录图的结点的状态。一个结点一共有三种状态：

  1. visited[i] == 0，结点i未被被搜索
  2. visited[i] == 1，结点i正在被搜索
  3. visited[i] == 2，结点i搜索完成

DFS的方法从结点的出度入手，如果一个结点搜索完毕就放入结果集，最终可以得到结果集xn,...,x3,x2,x1。则一个拓扑排序为x1,x2,x3,...,xn。因为本方法为DFS，所以拓扑排序中越往后的结点先被搜索完毕。

（Leetcode官方解法二）（广度优先搜索）解法二：本题使用广度优先搜索同样可以完成，定义edges保存图中所有边，定义remian[i]维护课程i需要学习的前驱课程的数量。定义队列que保存可以学习的课程。

以上过程为初始化的过程，现在说明搜索的过程，从队列中拿出一个元素u，edges[u]中的课程v即为以u为前驱课程的课程，remain[v]--，表示u已经学习，所有v的前置课程的数量减少一门，此时如果有remain[v] == 0，说明v的所有前置课程已经学习完毕，将v放入队列中。每从队列中拿出一个元素，说明该元素已经学习，visited++。当队列为空时，停止搜索，此时visited为可以拓扑排序的最大元素个数，如果visited == numCourses，说明该图存在拓扑排序，返回true，否则返回false。
与DFS方法不同，BFS课程学习的顺序符合人的直觉，故如果按照学习课程先后作为排序如x1,x2,x3,...,xn。则它们就是图的一个拓扑排序。
